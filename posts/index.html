# HTTP - a history Based on this great video HTTP/1 TCP/IP The first and most simple approach, making a TCP handshake for each resource which has to be shared. The downsides to this approach is that in order to speed up transfer rates, we want to send many resource pieces in parallel. With HTTP/1, this means doing a new TCP handshake everytime, which is costly and time consuming. This was alieviated in a later version HTTP/1.
